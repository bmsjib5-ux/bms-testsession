NHSO Single Claim API Spec v8.0

1 API Endpoints

  • Test: https://test.nhso.go.th/nhsoendpoint/
  • Production: https://nhsoapi.nhso.go.th/nhsoendpoint/

2 Workflow (27 Datasets)

  • Person Authen (1-3) - Authenticate patient, get authenCode
  • RealPerson (4-5) - Verify person status, get privilege info
  • NHSO Claim Detail (6-9) - Send/Cancel claim
  • HIS Payment Detail (10-11) - Send payment data
  • Inquiry (12-13) - Check payment status
  • Verify Privilege (16-17) - Verify privilege claim
  • Confirm Privilege (18-19) - Confirm privilege usage
  • Verify Payment (22-23) - Verify payment
  • Confirm Payment (24-25) - Confirm payment
  • Cancel Transaction (26-27) - Cancel by group/transaction ID

4 SQL View for HOSxP

Create a view bms_nhso_claim to query OPD visits:

CREATE OR REPLACE VIEW bms_nhso_claim AS
SELECT
  ov.vn,
  ov.vstdate AS visit_date,
  p.cid AS pid,
  p.pname AS title,
  p.fname AS first_name,
  p.lname AS last_name,
  p.birthday,
  CASE p.sex WHEN '1' THEN 'M' ELSE 'F' END AS sex,
  p.hn,
  p.hometel AS phone,
  ov.pttype,
  ptt.name AS pttype_name,
  COALESCE(ov.income, 0) AS total_amount,
  COALESCE(ov.paid_money, 0) AS paid_amount,
  COALESCE(ov.income - ov.paid_money, 0) AS privilege_amount,
  t.name AS province,
  ov.hospmain,
  ov.hospsub
FROM ovst ov
JOIN patient p ON p.hn = ov.hn
LEFT JOIN pttype ptt ON ptt.pttype = ov.pttype
LEFT JOIN thaiaddress t ON t.provincecode = p.province
WHERE ov.vstdate BETWEEN :dateFrom AND :dateTo;
Adjust column names to match your HOSxP schema. The view should return visit data needed for NHSO claims.
